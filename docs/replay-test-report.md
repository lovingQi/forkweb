# 叉车日志回放诊断工具测试报告

更新时间：2026-07-12

## 1. 测试目标

本次测试面向 `/replay` 叉车日志回放诊断工具，目标是确认工具在研发人员离线拿到现场日志和地图后，能够完成以下工作：

- 导入日志和地图并生成诊断会话。
- 展示一键概览、问题时间线、地图回放、错误码中心、任务视角和原始日志过滤。
- 支持规则型诊断知识库的新增、命中、禁用、导入导出和报告输出。
- 支持问诊助手的本地知识检索、相似案例推荐、在线模型配置与离线降级。
- 导出 Markdown/JSON 报告和诊断包时不泄漏 API Key 等敏感配置。
- 前后端构建与验证脚本可以稳定通过。

## 3. 测试流程

### 3.1 环境与服务检查

1. 检查 Git 工作区状态，确认测试前已有改动。
2. 检查前后端端口监听。
3. 检查后端健康接口与助手配置接口：
   - `GET /api/replay/assistant/config`
   - `GET /api/replay/assistant/status`
   - `GET /api/replay/session`

### 3.2 自动化验证

1. `npm run typecheck`
2. `npm run build`
3. `npm run replay:build`
4. `npm run replay:verify:samples`
5. `npm run replay:verify:knowledge`
6. `npm run replay:verify:assistant`

### 3.3 后端接口测试

1. 通过 `POST /api/replay/session` 加载样本日志和地图。
2. 验证概览接口包含时间范围、日志数量、车辆/版本信息、状态采样、任务、错误码、关键日志、根因和数据完整性。
3. 验证回放帧接口可返回位置、姿态、状态、任务和错误信息。
4. 验证事件时间线接口支持过滤和分页。
5. 验证错误码中心接口能返回定义、发生点和汇总。
6. 验证任务接口能返回任务列表和任务相关错误。
7. 验证原始日志接口支持等级、模块、错误码、任务、噪声和关键事件过滤。
8. 验证报告导出接口可输出 Markdown/JSON。
9. 验证诊断包导出不包含 API Key 或 `llm.local.json`。

### 3.4 知识库测试

1. 从原始日志证据创建临时知识规则。
2. 重新解析同一份日志，验证规则自动命中。
3. 验证命中进入诊断结论，并带有知识库来源、证据、置信度和处理办法。
4. 验证禁用规则后不再参与诊断。
5. 验证知识库导出和导入。
6. 验证 Markdown/JSON 报告包含知识库命中。
7. 测试结束后恢复原始知识库文件，避免测试残留。

### 3.5 问诊助手测试

1. 未配置 API Key 时，验证规则知识库仍可使用，问答进入离线降级。
2. 验证相似案例推荐能返回历史问题标题、处理办法和相似证据。
3. 验证上下文预览不会上传全量日志，只包含相关片段和摘要。
4. 使用 Mock OpenAI-compatible 服务验证在线问答链路。
5. 验证在线问答结果引用当前日志证据和知识库条目。
6. 验证 AI 建议不会自动写入知识库，只能作为草稿由人工确认。
7. 验证前端公开配置只返回 masked key。

### 3.6 前端可用性测试

1. 构建通过即确认 `/replay` 页面和组件类型正确。
2. 检查页面入口、Tab、按钮、进度条、问诊助手配置入口的代码绑定。
3. 启动前后端后保留服务运行，供人工继续访问。

### 3.7 收尾检查

1. 检查 `replay-server/config/llm.local.json` 不进入 Git。
2. 清理无关未跟踪缓存。
3. 确认知识库和本地模型配置没有测试残留。
4. 记录发现的问题、修复内容、回归结果和遗留风险。

## 5. 问题记录

### P1-001 知识库规则禁用验收失败

- 发现时间：2026-07-12
- 发现方式：执行 `npm run replay:verify:knowledge`
- 现象：脚本在“应能禁用知识规则”断言处失败。
- 定位：首次运行时同时执行了 `replay:verify:knowledge` 和 `replay:verify:assistant`，两个脚本都会临时写入并恢复 `replay-server/config/knowledge-base.json`，存在测试过程互相覆盖。
- 复测：单独重跑 `npm run replay:verify:knowledge` 后通过，禁用规则、重新解析不命中、导入导出均满足验收。
- 结论：非产品代码缺陷，归类为测试执行注意事项；后续验收脚本应避免并行操作同一个知识库文件。
- 状态：关闭。

### N-001 JSON 报告字段断言修正

- 发现时间：2026-07-12
- 发现方式：后端接口测试脚本最初断言 `/api/replay/report.json` 必须包含完整 `events`。
- 定位：当前报告导出的契约是 `keyTimeline`，保留最多 100 条关键事件摘要，避免 JSON 报告过大；完整事件通过 `/api/replay/events` 查询。
- 结论：测试断言按现有接口契约修正为 `overview + keyTimeline + knowledgeMatches`。
- 状态：关闭。

### N-002 Playwright 浏览器内核缺失

- 发现时间：2026-07-12
- 发现方式：尝试执行 `/replay` 页面浏览器级截图冒烟测试。
- 现象：Playwright CLI 可运行，但本机缺少 `chromium_headless_shell`，提示需要安装浏览器内核。
- 处理：已安装 Playwright Chromium 和 headless shell，并新增真实浏览器点击链路测试。
- 状态：关闭。

### N-003 E2E 定位器稳定性修正

- 发现时间：2026-07-12
- 发现方式：执行 `npm run test:e2e:replay`。
- 现象：Element Plus Tab 会保留隐藏面板 DOM，导致“错误码”“关键词”等重复/隐藏输入框影响 Playwright 严格定位。
- 处理：E2E 用例改为限定当前可见 Tab 面板；原始日志链路保留真实点击“过滤”按钮，不强制填写关键词。
- 状态：关闭。

### N-004 实时车端接口控制台噪声

- 发现时间：2026-07-12
- 发现方式：执行 `npm run test:e2e:replay`。
- 现象：页面打开后实时监控接口和 WebSocket 可能输出 `/api/state`、`/api/map`、`/api/params`、`/ws/high`、`/ws/low` 相关 500 或握手失败日志。
- 结论：这些属于当前无实车/无实时车端时的既有噪声，不影响 `/replay` 回放诊断链路；E2E 中已过滤。
- 状态：关闭。

## 7. 最终结论

本轮测试结论：通过。

已确认工具当前具备离线日志诊断、地图回放、错误码/任务/时间线分析、规则知识库、问诊助手、报告导出和诊断包导出的基础可用性。自动化验收、后端接口验收和前端构建均通过。

本轮新增了真实浏览器 E2E 测试能力，测试中发现的问题均为测试执行方式或测试环境限制：

- 知识库验收脚本和助手验收脚本不能并行运行，因为它们都会临时写入同一个 `knowledge-base.json`。
- JSON 报告当前导出的是精简 `keyTimeline`，完整事件应通过 `/api/replay/events` 查询。
- Playwright Chromium 浏览器内核已安装，`npm run test:e2e:replay` 已通过。

遗留风险：

- 样本日志中未解析到有效任务段，本轮任务视角只验证了接口和空态，仍需要有真实任务 ID 的日志继续覆盖。
- 当前知识库和助手脚本使用共享配置文件，后续若纳入 CI，应改为隔离临时知识库路径或串行执行。
