# forkweb 现场操作指南

> 一页纸快速上手指南，覆盖登录、建单、上传日志、查看排查向导、执行步骤、升级研发六个核心动作。

## 1. 登录系统

1. 打开浏览器，访问系统地址（如 `http://192.168.1.100:8091`）。
2. 输入账号密码（默认管理员：`admin / admin123`，首次登录后请修改密码）。
3. 点击「登录」。

**截图占位**：登录页面，填写账号密码后点击登录按钮。

## 2. 新建工单

1. 登录后默认进入「工单管理」列表页。
2. 点击右上角「新建工单」按钮。
3. 填写：
   - **标题**：一句话概括问题，例如「3 号车定位丢失」。
   - **项目现场**：选择事发车辆所属现场。
   - **描述**：补充现象、时间、影响范围。
   - （可选）发生时间、影响程度。
4. 点击「提交工单」。

**截图占位**：新建工单表单，标题/现场/描述已填写。

## 3. 上传日志

1. 进入工单详情页后，点击「上传日志」区域或详情页「补充上传」按钮。
2. 选择车辆导出的日志文件（支持 `.log`、`.zip`、`.tar.gz`，单次总大小不超过 200MB）。
3. 点击「上传」。
4. 系统会自动开始分析，状态显示为「分析中」。

**截图占位**：文件选择弹窗与上传进度。

## 4. 查看排查向导

1. 分析完成后，状态变为「待现场排查」。
2. 页面中部展示「排查向导」，按优先级列出 Top 3 排查路径。
3. 默认展开最高优先级路径，查看步骤标题、操作说明与判断标准。
4. 点击步骤可展开证据上下文（日志片段、错误码、时间线）。

**截图占位**：排查向导面板，最高优先级路径已展开。

## 5. 执行步骤并记录状态

1. 按向导顺序执行每一步：
   - **未检查**：尚未执行。
   - **已通过**：步骤已确认无问题。
   - **未通过**：步骤发现问题，继续按提示处理。
   - **不适用**：该步骤无法在本场景执行，必须选择原因。
2. 首次执行「现场操作」类步骤前，需勾选安全确认。
3. 状态变更会自动记录操作人与时间戳。

**截图占位**：步骤单选按钮与安全确认弹窗。

## 6. 升级研发

当所有关键步骤执行完毕仍无法解决，或遇到缺少权限/工具的情况：

1. 点击「需要研发介入」按钮。
2. 选择升级原因：
   - 按向导排查仍未解决
   - 缺少权限或工具
   - 疑似软件缺陷
   - 诊断结论不可信
   - 需要远程协助
   - 其他
3. 提交后工单状态变为「待研发介入」，系统会自动通知研发。

**截图占位**：升级研发对话框，选择原因后点击提交。

## 常见问题

| 问题 | 处理建议 |
|------|----------|
| 上传后状态卡在「分析中」 | 耐心等待，分析超时为 10 分钟；超时后状态会回退并记录事件。 |
| 排查向导显示「未匹配到已验证规则」 | 当前日志未命中知识库，可尝试补充上传更多日志后升级研发。 |
| 误提交工单 | 若为提单人，可点击「取消工单」废弃。 |
| 需要修改工单标题/描述 | 未终结前点击「编辑基本信息」修改。 |

## 安全提示

- 执行现场操作步骤前，务必确认车辆已停止或处于安全模式。
- 不要在公共网络中传播默认管理员密码。
- 遇到无法判断的情况，优先选择升级研发，不要强行操作。
